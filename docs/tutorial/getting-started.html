Getting Started

Everything you need to install DOE and Claude Code — even if you've never used a terminal before.

Prerequisites

Before you can use DOE, you need three things set up on your machine. Each card below explains what it is, why you need it, and how to get it.

VSCode

Visual Studio Code is a free code editor. You'll use it to see your project files and run Claude Code. Download from code.visualstudio.com. It has a built-in terminal so you don't need a separate app.

Git

Version history for code, like Google Docs version history but for files. Every time you finish work, you "commit" it — like saving your game. Download from git-scm.com. Mac users: it may already be installed.

Claude Code

The AI coding assistant that powers DOE. You need a Claude Pro or Max subscription. Install via the VSCode marketplace or the Claude Code docs. No API key needed — authentication is automatic.

ⓘ
Opening your terminal: The easiest option is VSCode's built-in terminal — open VSCode and go to Terminal > New Terminal. On Mac, press Cmd+Space, type "Terminal", hit Enter. On Windows, search for "Windows Terminal" in the Start menu.

Choose Your Setup

There are two paths to get DOE running. Both give you the same result — a project folder with all the configuration, commands, and scripts that make DOE work.

Start from DOE Starter Kit (Recommended)

Clone the starter kit repository and run the setup script. This gives you everything pre-configured — commands, hooks, directory structure, and a solid default CLAUDE.md.

Add DOE to an Existing Project

If you already have a project, create the DOE files manually: CLAUDE.md, directives/, execution/, tasks/todo.md, STATE.md, and learnings.md.

Option A: Clone the Starter Kit

  1. 1

    Clone the repository

    Open your terminal and run:

    git clone https://github.com/iPolyphian/doe-starter-kit.git my-project
cd my-project

    Replace my-project with whatever you want your folder to be called.

  2. 2

    Run the setup script

    bash setup.sh

    This installs slash commands (/stand-up, /wrap, etc.), sets up Git hooks, creates the directory structure DOE expects, and initialises Git if needed.

  3. 3

    Verify it works

    claude

    Claude Code's interface should appear. Type /commands to confirm the DOE commands are available. You should see /stand-up, /wrap, and /sitrep in the list.

Option B: Add DOE to an Existing Project

If you already have a project folder with code in it, create these files and directories:

  • CLAUDE.md — the project instruction file (copy from the starter kit or write your own)
  • directives/ — folder for SOP documents
  • execution/ — folder for deterministic Python scripts
  • tasks/todo.md — task tracker
  • STATE.md — session memory
  • learnings.md — project learnings log

Configure Your Project

CLAUDE.md — The Project Instruction File

CLAUDE.md is the single most important file in DOE. It's the instruction manual that Claude reads at the start of every conversation. It controls how Claude behaves: what rules it follows, what it's allowed to do, and where it puts files.

The starter kit gives you a solid default. Before your first real session, customise these parts:

  • Directory Structure — must match your actual project folders. Claude uses this to know where to create new files.
  • Progressive Disclosure triggers — tell Claude "when you're about to do X, read file Y first." Add 2–3 for your project's key areas.
  • Project-specific guardrails — rules that protect your project (e.g., "never delete user data without confirmation").
ⓘ
The 200-line rule: Keep CLAUDE.md under ~200 lines. Claude reads the whole thing at the start of every conversation, and every line uses context window space. If something needs more detail, put it in a directive file and add a trigger.

Directory Structure

After setup, your project has this structure. You don't need to memorise it — Claude knows where everything goes.

my-project/
  CLAUDE.md          # Project instruction file (Claude reads this first)
  STATE.md           # Session memory (updated by /wrap)
  learnings.md       # Project learnings log (grows over time)
  directives/        # SOP documents for specific tasks
  execution/         # Deterministic Python scripts
  tasks/
    todo.md          # Task tracker with contracts
  .env               # API keys and credentials (NEVER commit)
  .claude/           # Hooks, settings, plans
  .githooks/         # Git hooks for quality checks

API Keys and .env

If your project uses external APIs, store the keys in a .env file in the project root. This file is automatically excluded from Git so your secrets stay safe. Claude Code reads .env automatically when running execution scripts.

Your First Command

With everything installed and configured, start Claude Code and run your first DOE command:

Terminal 
$ claude
# Claude Code opens. You type:
/stand-up
# Claude reads your project state and shows a status card.
# On your first session, most fields will be empty — that's normal.
