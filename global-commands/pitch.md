Read CLAUDE.md, ROADMAP.md, STATE.md, tasks/todo.md, and learnings.md to understand the product, what's been built, and where it's heading.

Then generate 3–5 genuinely useful ideas for the project. Don't pad — if you only have 2 good ones, pitch 2.

For each idea, pitch it. Present each one like this:

```
### [Idea Name]
**Size:** Small / Medium / Large
**Type:** Feature | Enhancement | Infrastructure | Data | UX
**Value pitch:** [1-2 sentences — why this matters for the product. What does it unlock? Who benefits? Be honest about impact, not salesy.]
**Effort sketch:** [1 sentence — what's roughly involved to build it]
```

After presenting all pitches, ask the user which ones (if any) they want added to ROADMAP.md under ## Ideas. Only add the ones they approve — don't add anything without sign-off.

When adding to ROADMAP.md, use this format:
```markdown
### [Idea Name]
[One-line description.] *(pitched DD/MM/YY)*
```

Rules:
- Don't pitch things already on the roadmap or in Done.
- Don't pitch things that are just "nice to have" refactors unless they genuinely unblock something.
- Be honest about size. If something is Large, say so — don't undersell the effort.
- Think about what the product actually needs at this stage, not what would be cool in theory.
- Genuine > impressive. A small idea that solves a real gap beats a flashy feature that doesn't.
